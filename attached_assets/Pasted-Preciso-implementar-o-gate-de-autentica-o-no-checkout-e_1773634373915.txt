TAREFA 1 — Gate de autenticação no carrinho
No componente da página /cart (procurar em client/src/pages/):

Ao clicar no botão "Finalizar compra" ou "Ir para o checkout":
  - Verificar se o usuário está autenticado via useAuth()
  - Se NÃO estiver logado: exibir um modal elegante de 
    autenticação (não redirecionar para /login — manter no carrinho)
  - Se estiver logado: ir direto para /checkout normalmente

O modal de autenticação deve ter:
  - Título: "Para finalizar sua compra"
  - Duas abas: "Entrar" e "Criar conta"
  - Aba "Entrar": formulário com e-mail e senha + botão "Entrar"
  - Aba "Criar conta": formulário com e-mail, senha e aceite 
    de termos + botão "Criar conta e continuar"
  - Após login ou cadastro bem-sucedido: fechar o modal e 
    redirecionar automaticamente para /checkout
  - O carrinho (localStorage) deve ser preservado integralmente
    durante todo o processo
  - Estilo visual consistente com o restante do site
  - Botão X para fechar o modal (volta ao carrinho sem autenticar)

TAREFA 2 — Pré-preenchimento do checkout com dados do usuário
No componente da página /checkout (procurar em client/src/pages/):

Ao carregar a página /checkout, se o usuário estiver logado:
  - Buscar dados via GET /api/users/profile
  - Pré-preencher automaticamente os campos da etapa 1:
    * E-mail → campo email
    * full_name → campo nome completo
    * cpf_cnpj → campo CPF/CNPJ
    * phone → campo telefone
    * address_zip → campo CEP
    * address_street → campo logradouro
    * address_number → campo número
    * address_complement → campo complemento
    * address_neighborhood → campo bairro
    * address_city → campo cidade
    * address_state → campo estado
  - Campos pré-preenchidos devem ser editáveis 
    (o usuário pode corrigir antes de confirmar)
  - Se um campo estiver vazio no perfil, deixar em branco 
    normalmente para o usuário preencher

TAREFA 3 — Salvar dados na conta após checkout
Após o usuário confirmar os dados na etapa 1 do checkout

  - Chamar PATCH /api/users/profile com os dados preenchidos
  - Salvar: full_name, cpf_cnpj, phone, address_street,
    address_number, address_complement, address_neighborhood,
    address_city, address_state, address_zip
  - Fazer isso de forma silenciosa (sem bloquear o fluxo)
  - Se a chamada falhar, apenas logar o erro — não impedir 
    o usuário de continuar para o pagamento
  - Na próxima compra, esses dados já estarão pré-preenchidos
