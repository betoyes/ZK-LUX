MELHORIA 1 — Invalidar sessões após reset de senha (CRÍTICO)
Arquivo: server/routes.ts
Localizar: a rota POST /api/auth/reset-password
Após a linha que chama storage.updateUserPassword(), adicionar código que 
deleta todas as sessões ativas do usuário no banco de dados PostgreSQL usando 
a tabela "session" que já existe (usada pelo connect-pg-simple).
A query deve ser:
  DELETE FROM session WHERE sess::jsonb -> 'passport' -> 'user' = to_jsonb($1::int)
Parâmetro: tokenData.userId
Fazer isso antes de retornar o JSON de sucesso.
Validar: após reset, o usuário deve ser deslogado de todos os dispositivos.

MELHORIA 2 — bcrypt com custo uniforme (custo 12 em todos os pontos)
Arquivo: server/routes.ts
Localizar: TODAS as chamadas bcrypt.hash() no arquivo
Substituir custo 10 por 12 em: rota de registro (/api/auth/register) e 
rota de reset (/api/auth/reset-password).
A rota /api/auth/change-password já usa 12 — manter como está.
Validar: buscar no arquivo por "bcrypt.hash" e confirmar que todos usam 10 → 12.

MELHORIA 3 — Aumentar expiração do token de reset de senha para 3 horas
Arquivo: server/routes.ts
Localizar: rota POST /api/auth/forgot-password
Encontrar a linha: Date.now() + 60 * 60 * 1000
Substituir por: Date.now() + 3 * 60 * 60 * 1000
Validar: token de reset deve expirar em 3 horas, não 1 hora.

Validar: simular falha de envio e confirmar que o admin recebe notificação.

MELHORIA 5 — Aumentar expiração do token de verificação de e-mail para 48 horas
