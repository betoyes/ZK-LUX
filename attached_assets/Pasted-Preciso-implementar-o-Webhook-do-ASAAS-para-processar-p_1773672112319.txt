TAREFA 2 — Criar endpoint do Webhook
No arquivo server/routes.ts, adicionar ANTES de todas as
outras rotas (para garantir que o body raw chegue corretamente):

Endpoint: POST /api/webhooks/asaas

Comportamento:
1. Validar o token de autenticação no header:
   header: asaas-access-token
   Comparar com process.env.ASAAS_WEBHOOK_TOKEN
   Se não bater: retornar 401 imediatamente
   Se ASAAS_WEBHOOK_TOKEN não estiver configurado:
   logar aviso mas processar mesmo assim (ambiente dev)

2. Extrair do body:
   - event: string (ex: "PAYMENT_CONFIRMED", "PAYMENT_RECEIVED")
   - payment.id: string (asaasPaymentId)
   - payment.status: string
   - payment.paymentDate: string (pode ser null)

3. Processar apenas os eventos:
   - PAYMENT_CONFIRMED
   - PAYMENT_RECEIVED
   Ignorar todos os outros eventos silenciosamente
   (retornar 200 para não interromper a fila do ASAAS)

4. Idempotência — CRÍTICO:
   Antes de processar, verificar se este asaasPaymentId
   já foi processado com status CONFIRMED ou RECEIVED.
   Se já foi processado: retornar 200 sem fazer nada.
   Isso evita pedidos duplicados se o ASAAS enviar o
   mesmo evento mais de uma vez.

5. Ao receber PAYMENT_CONFIRMED ou PAYMENT_RECEIVED:
   a) Buscar o pagamento local pelo asaasPaymentId
      via storage.getAsaasPaymentByAsaasId(asaasPaymentId)
      Se não encontrar: retornar 200 (pagamento não é deste sistema)

   b) Atualizar o status do pagamento local:
      storage.updateAsaasPayment(localPayment.id, {
        status: event === 'PAYMENT_RECEIVED' ? 'RECEIVED' : 'CONFIRMED',
        paymentDate: payment.paymentDate || new Date().toISOString().split('T')[0]
      })

   c) Buscar o pedido associado ao pagamento:
      Procurar na tabela orders pelo asaas_payment_id
      Se encontrar: atualizar status para 'confirmed'
      Se não encontrar: criar um novo pedido com:
        - orderId: 'ZK-' + Date.now() + '-' + Math.random().toString(36).substr(2,4).toUpperCase()
        - status: 'confirmed'
        - userId: localPayment.userId (pode ser null)
        - total: localPayment.value
        - customer: buscar nome do cliente via asaas_customers
        - date: data atual

   d) Disparar e-mail de confirmação para o cliente:
      Buscar e-mail do cliente via asaas_customers
      Chamar sendOrderConfirmationEmail() — ver TAREFA 3

   e) Retornar HTTP 200 com { received: true }

6. Em caso de erro interno: retornar 200 mesmo assim
   (o ASAAS requer 200 para não pausar a fila)
   Logar o erro detalhado no console para debugging
